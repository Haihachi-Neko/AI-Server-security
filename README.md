# AI-Server-security（開発中止）

## プロジェクト概要
LLMを活用し、AIによってサーバーに不審なプロセスなどがないか監視するプログラムです。

## 使用方法
Python、pipのインストール済み、このリポジトリのクローン済みを前提として説明致します。
1. python環境のセットアップ
    python 3.14環境にて動作確認済みです。
2. ollamaのインストール
    https://ollama.com/ よりollamaをインストールしてください。
3. AIモデルのインストール
    ```Bash
    ollama pull qwen3.5:2b
    ```
    を実行し、AIモデルをインストールしてください。
4. ライブラリのインストール
    （venvを使用することを推奨します。）
    pip install -r requirements.txtを実行し、ライブラリをインストールしてください。
5. 実行
    ```Bash
    python main.py
    ```
    または
    ```Bash
    python3 main.py
    ```
    で実行します。

※環境によって異なります。臨機応変に対応しセットアップしていただきますようお願いいたします。

## 追記
そもそもの構造としてはローカルLLMを使用予定でしたが、私の所有するCUDAがollamaで使用できるグラフィックボードがQuadro P620しかなく、実用できるレベルのパラメーター数のモデルを使用すると生成時間が極めて長くなり開発が困難であるためこのプロジェクトは中止となりました。  
今後の方針としては、LLMを使用するものでなく教師なし学習を用いたサーバー監視専用のモデルを作成し、それを用いたサーバー監視ツールを作成する予定です。また性能の良いグラフィックボードを入手できた際は、このソフトウェアの開発を再開する可能性があります。  
アイデア自体は良かったと思うので、プログラムをもっとしっかりと作成（より多いログのLLMへの提供やLLMからツールを呼び出し、情報を提供するなど）できればと思っていたところですが、頓挫したためソフトウェアの開発が中途半端なところで開発中止となります。なんかすみませんm(_ _)m
